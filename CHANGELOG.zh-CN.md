## 0.1.0 - 2026-07-23

### 新增

- 建立 Unimail 简体中文桌面应用的工程基础、原生构建检查与未签名测试安装包产出流程。
- 增加本地邮件领域模型、SQLCipher 加密数据库、系统凭据保护和数据库健康检查基础能力。
- 桌面状态栏现在会显示经过安全解码的本地加密存储就绪状态和数据库版本。
- 增加跨邮箱统一 Provider 合同、RFC 邮件解析与组合基础，并确保含糊的发送结果不会被自动重试。
- 增加可崩溃恢复的同步、已读状态合并和离线发送复核基础；恢复网络后只提示重新确认，不会自动投递草稿。
- 增加 Gmail 系统浏览器安全连接、账号重连和同步接入界面；未配置公开客户端 ID 的测试安装包仍可正常使用其他本地功能。
- 增加 Outlook.com 与 Microsoft 365 工作/学校账户连接，支持最新 500 封收件箱、增量同步、已读状态、附件以及 MIME 发送和回复。
- 增加 QQ 邮箱与 163 邮箱授权码连接，可通过加密 IMAP/SMTP 同步最新邮件、更新已读状态、下载附件并安全发送邮件；发送结果不明确时不会自动重发。
- 增加跨账户统一收件箱和本地邮件阅读器，支持账户与未读筛选、连续分页、键盘切换、延迟自动已读以及离线缓存阅读；HTML 邮件默认阻止远程内容并隔离活动代码，可为当前邮件临时安全显示图片，外部链接会显示真实地址后再次确认。
- 增加纯文本写信与单一“回复”入口，本地草稿会在停止编辑 1 秒后自动保存，并可从“草稿”重新打开；“已发送”会区分等待邮箱确认、已对账和发送结果待确认，离线或结果不明确时不会自动重发。
- 增加安全附件另存与离线邮件搜索：附件会流式保存到用户选择的位置并显示进度、取消和失败状态，成功后不保留应用私有副本；可按标题、正文和发件人在全部或指定账户的本地收件箱中搜索中文及其他 Unicode 内容。
- 增加“安全与诊断”界面，可在本机查看应用、联网、加密存储和各邮箱服务商的计数状态；诊断不会自动上传，也不包含邮箱地址、邮件内容或本地路径。
- 增加可验证的 Windows x86_64 与 macOS Universal 版本发布流程，正式下载会同时提供中文说明、SHA-256 校验值和构建来源；未完成生产签名或 Apple 公证的平台会被准确标为测试预发布版。

### 变更

- 添加账户窗口现在可选择 Gmail 或 Outlook，并会为需要重新授权的账户保留对应的重新连接入口。
- “收件箱”“草稿”“已发送”现在是可切换的真实视图；手动刷新“已发送”会只读检查邮箱服务中的 Sent 记录，并按稳定邮件标识合并本地等待项。
- V1 版本保持手动下载，不启用应用内自动更新，也不会发布可被误认为安全更新通道的空签名或 updater 元数据。
- 附件保存失败后会明确显示“重试”，并补充系统原生另存与断网本地搜索的所有者验收步骤。

### 修复

- 修复未签名测试安装包在部分环境中启动后立即退出的问题，并在 Windows 与 macOS 打包后自动验证应用能够保持运行。
- 修复持续集成随 Rust 稳定版自动升级后可能中断的问题，提高 Windows 与 macOS 测试安装包的构建稳定性。
- 修复 Gmail 连接成功后再次打开账户窗口无法重新连接，以及授权失效后账户入口消失的问题。
- 修复 macOS 网络边界校验可能误判安全断开的问题，避免无签名测试安装包被错误阻塞。
- 改进跨平台测试安装包的校验反馈，构建失败时可准确定位具体前端检查、Rust 组件或邮件 Provider 测试用例。
- 修复 Graph 请求头名称大小写差异可能误阻塞 Windows 与 macOS 测试安装包的问题。
- 修复取消附件下载后界面可能仍停留在下载进度状态的问题。
