# 更新日志

本文件记录 Unimail 面向用户的变化。版本发布前，将“未发布”中的条目移动到对应版本章节。

## 未发布

### 新增

- 建立 Unimail 简体中文桌面应用的工程基础、原生构建检查与未签名测试安装包产出流程。
- 增加本地邮件领域模型、SQLCipher 加密数据库、系统凭据保护和数据库健康检查基础能力。
- 桌面状态栏现在会显示经过安全解码的本地加密存储就绪状态和数据库版本。
- 增加跨邮箱统一 Provider 合同、RFC 邮件解析与组合基础，并确保含糊的发送结果不会被自动重试。
- 增加可崩溃恢复的同步、已读状态合并和离线发送复核基础；恢复网络后只提示重新确认，不会自动投递草稿。
- 增加 Gmail 系统浏览器安全连接、账号重连和同步接入界面；未配置公开客户端 ID 的测试安装包仍可正常使用其他本地功能。
- 增加 Outlook.com 与 Microsoft 365 工作/学校账户连接，支持最新 500 封收件箱、增量同步、已读状态、附件以及 MIME 发送和回复。

### 变更

- 添加账户窗口现在可选择 Gmail 或 Outlook，并会为需要重新授权的账户保留对应的重新连接入口。

### 修复

- 修复持续集成随 Rust 稳定版自动升级后可能中断的问题，提高 Windows 与 macOS 测试安装包的构建稳定性。
- 修复 Gmail 连接成功后再次打开账户窗口无法重新连接，以及授权失效后账户入口消失的问题。
- 修复 macOS 网络边界校验可能误判安全断开的问题，避免无签名测试安装包被错误阻塞。
- 改进跨平台测试安装包的校验反馈，构建失败时可准确定位具体前端检查、Rust 组件或邮件 Provider 测试。
