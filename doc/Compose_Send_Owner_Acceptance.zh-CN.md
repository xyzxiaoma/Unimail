# 写信、草稿与已发送所有者验收清单

本清单用于仓库所有者使用专门的测试邮箱验证 Unimail V1 写信链路。请勿在 issue、日志、截图或提交中记录真实邮箱地址、主题、正文、Message-ID、服务商邮件 ID、授权码或令牌。

## 准备

1. 至少连接一个测试账户；跨服务商验收时分别准备 Gmail、Outlook、QQ 和 163 测试账户。
2. 只向自己控制的测试收件箱发送虚构内容，不使用生产联系人或真实业务邮件。
3. 打开服务商网页端或官方客户端，以便独立检查 Inbox 与 Sent/已发送文件夹。

## 新邮件与本地草稿

- [ ] 点击“写邮件”可选择一个已连接账户，填写收件人、可选抄送/密送、主题和纯文本正文。
- [ ] 停止编辑约 1 秒后显示已保存；关闭或按 Escape 后，可在“草稿”中重新打开相同内容。
- [ ] 打开后不输入任何内容并关闭，不会创建空白草稿。
- [ ] 删除草稿需要二次确认，且不会删除服务器邮件。
- [ ] 无收件人、地址无效或主题与正文同时为空时无法发送；仅主题为空时需要再次确认。
- [ ] 密送收件人能收到邮件，但收到的 MIME 头中不出现 Bcc。

## 回复

- [ ] 阅读一封收到的邮件时只显示“回复”，不显示“回复全部”。
- [ ] 回复自动使用原邮件所属账户，发件账户不可切换，只预填原始发件人。
- [ ] 回复主题只有一个 `Re:`，正文引用为纯文本；服务商网页端显示在线程或会话中的正确位置。

## 离线与不明确结果

- [ ] 断网后点击发送只保存草稿并提示未发送；恢复网络不会自动投递。
- [ ] 联网后重新打开草稿，必须再次检查并明确确认才会发送。
- [ ] 若测试环境能模拟提交后断线，记录会显示“发送结果待确认”，再次点击不会直接重发。
- [ ] 手动刷新“已发送”后，只有在网页端仍找不到邮件并接受重复风险提示时，才可解锁一次再次发送。

## Sent 对账

- [ ] Gmail：发送后等待项可通过 Gmail message ID 或 RFC Message-ID 且带 `SENT` 标签的邮件完成对账。
- [ ] Outlook：发送后等待项可通过 Sent Items 中相同 internet Message-ID 的邮件完成对账。
- [ ] QQ/163：只在已发现的 Sent/已发送文件夹中按 Message-ID 查询；Unimail 不额外 APPEND 邮件副本。
- [ ] 邮箱已接受但远端记录尚未出现时，Unimail 显示“等待邮箱确认”，并允许查看本地发送内容，但不伪造服务器邮件。
- [ ] 刷新后匹配成功时，等待项变为“已发送”，列表中只保留一个权威记录。
- [ ] 重启应用后，等待确认、已对账和结果待确认状态仍然保持。

验收问题只记录平台、应用版本、邮箱服务商、操作阶段和固定错误码。QQ/163 的真实账号任务必须保持进行中，直到所有者完成其独立验收清单。
