# 附件与本地搜索所有者验收清单

本清单用于仓库所有者通过专门的测试邮箱验证系统原生附件另存和离线本地搜索。请只使用虚构邮件与无敏感信息的测试附件；不要在 issue、日志、截图或提交中记录真实邮件内容、附件内容、搜索词、本地保存路径、账户地址或服务商标识。

## 准备

1. 连接一个专门的测试账户，并同步至少两封 Inbox 邮件。
2. 测试邮件应覆盖标题、纯文本正文、发件人名称和发件人地址中的虚构关键词，并包含一字、两字和多字中文关键词。
3. 准备两个无敏感信息的小附件，其中至少一个邮件同时包含两个附件；另准备一个接近但不超过 32 MiB 上限的附件。
4. 在发送测试附件前记录其本地 SHA-256，仅保留哈希，不记录附件内容或最终保存路径。

## 系统原生另存

- [ ] 点击收到邮件中的“保存”，由操作系统打开原生另存窗口；应用界面不显示所选目录或完整路径。
- [ ] 取消原生另存窗口后，附件恢复为“保存”，不显示失败提示，也没有创建目标文件或 Unimail 部分文件。
- [ ] 路径形式、控制字符、Windows 保留名和尾随点/空格不会作为危险建议文件名出现；仍能得到可识别的安全名称。
- [ ] 保存小附件时可看到独立进度，完成后显示“已保存”；目标位置只有一个完整文件，SHA-256 与发送前记录一致。
- [ ] 同一封邮件的两个附件可分别操作；其中一个下载、取消或失败不会禁用另一个附件。
- [ ] 下载中点击“取消”后显示“下载已取消”，目标位置没有完整文件或残留部分文件，并可再次点击“保存”。
- [ ] 目标位置已有同名文件时，原文件内容不变，Unimail 显示冲突提示和“重试”，不会静默覆盖。
- [ ] 断网或临时阻断邮箱服务后保存附件，会显示可理解的安全错误和“重试”；恢复网络后可以重新选择位置并成功保存。
- [ ] 下载接近上限的附件可以完成；超过 32 MiB 或服务商报告大小与实际内容不一致时，不生成可误认为完整的目标文件。
- [ ] 下载过程中强制结束应用，再次启动后不会保留 Unimail 所有的部分文件；用户此前明确保存成功的文件仍然存在。
- [ ] 删除测试账户后，应用自有的附件缓存/清理项被清除，但用户明确另存到其他位置的文件不被删除。

## 断网本地搜索

- [ ] 完成同步后断开网络，确认邮箱服务不可访问，但应用仍能打开已缓存的 Inbox 邮件。
- [ ] 分别按标题、正文、发件人名称和发件人地址中的虚构关键词搜索，均能返回对应本地 Inbox 邮件。
- [ ] 中文一字、两字和多字关键词，以及拉丁字母大小写、标点和 Unicode 内容能够得到可预测结果或安全的空结果。
- [ ] 输入引号、星号、`OR`、`NEAR`、括号和混合标点不会暴露 SQLite/FTS 错误，也不会让应用退出。
- [ ] “全部账户”和单账户范围只返回对应邮件；“未读”筛选只返回本地仍为未读的匹配邮件。
- [ ] 非 Inbox 邮件、已禁用账户和正在删除的账户不会出现在结果中。
- [ ] 搜索结果显示安全的匹配摘要，点击后复用现有阅读器打开正确的本地邮件。
- [ ] 当匹配结果超过 50 条时可继续加载，前后页无重复、漏项或顺序跳动。
- [ ] 快速连续更换关键词时，旧查询结果不会覆盖新查询；清除搜索后立即恢复原收件箱与原有账户/未读范围。
- [ ] 整个断网搜索过程不要求重新连接 Provider，也不触发远端搜索。

## 验收记录

只记录应用版本、操作系统、邮箱服务商、验收项目结果和固定错误码。发现问题时使用新的虚构关键词与附件重现，不提交数据库、凭据、邮件导出文件、附件样本、截图中的真实内容或本地路径。
